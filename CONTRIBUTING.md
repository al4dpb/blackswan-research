# Contributing to Black Swan Research

Thank you for considering a contribution. This is a research archive — the standard is primary-source citation, evidence-tier discipline, and respect for the bilingual ES/EN structure.

## Ground rules

1. **Cite primary sources.** Wikipedia is not a source; it's a finding aid. Link to the underlying paper, archive, government document, or first-hand account.
2. **Mark evidence honestly.** Use `CORROBORATED`, `CONTESTED`, `OPEN-CASE`, or `SPECULATIVE` (see [EVIDENCE-RUBRIC.md](./EVIDENCE-RUBRIC.md)). Overclaiming is worse than missing data.
3. **Disclose AI assistance.** If you used an LLM to draft, summarize, or translate, say so in your PR description. Submissions silently generated by AI without verification will be closed.
4. **Stay bilingual.** Articles have `en.mdx` and `es.mdx`. Update both, or open an issue asking a translator to follow up.
5. **No promotional content.** This repo is research. If you want to plug a product, course, or app, it doesn't belong here.

## Types of contributions

### 1. Citation fix

Broken URL, wrong year, misattributed quote, missing DOI.

- File path: the article's `en.mdx` and/or `es.mdx`
- PR title: `fix(citation): <article-slug> — <what>`
- Body: paste before/after of the changed line plus a link to the correct source.

### 2. Corroborating source

You found another peer-reviewed study (or replication, or government archive document) that strengthens an existing claim.

- File path: the article's `meta.json` (`sources` array) AND the relevant `<AnswerCapsule source="...">` line in the MDX.
- PR title: `add(source): <article-slug> — <source-shorthand>`
- Body: full citation (authors, year, journal, DOI/URL), one sentence on how it corroborates.

### 3. Replication report

You ran a protocol yourself or with a group. Single subject or n-of-many both welcome.

- File path: create `replications/<article-slug>/<your-handle>-<YYYY-MM-DD>.md`
- Template (copy this):

```markdown
---
contributor: <your handle or pseudonym>
date: <YYYY-MM-DD>
protocol: <article-slug this replicates>
n: <number of subjects>
duration: <e.g., 14 days>
evidence_tier: <SELF-REPORT | INSTRUMENTED | LAB-CONDITIONS>
---

## Protocol followed
<exact steps you ran, deviations from the article>

## Measurements
<HRV, sleep score, journal entries, etc. Raw data preferred.>

## Result
<plain-language summary>

## Limitations
<what would invalidate this>

## Sources
<any references>
```

- PR title: `replication: <article-slug> — n=<N> — <contributor>`

### 4. Translation polish

Improve fluency, accuracy, or terminology consistency between `en.mdx` and `es.mdx`.

- PR title: `i18n: <article-slug> — <en|es>`
- Body: note specific phrases changed and why.

### 5. New article proposal

**Open an issue first** using the `new-research.md` template. We'll triage before you write to avoid duplicate effort. After green-light, follow the article structure of an existing piece (e.g., [`articles/jacobo-grinberg/`](./articles/jacobo-grinberg)).

## PR checklist

Before submitting:

- [ ] I cited primary sources (not Wikipedia-only).
- [ ] I marked the appropriate evidence tier.
- [ ] I disclosed any AI assistance.
- [ ] If editing an article, I updated both `en.mdx` and `es.mdx` (or opened an issue for the missing language).
- [ ] I preserved `<AnswerCapsule>` and other semantic markup.
- [ ] I am submitting under [CC BY-SA 4.0](./LICENSE) and confirm I have the right to do so.

## Review process

- Citation fixes & translation polish: usually merged within a week.
- Corroborating sources: reviewed for source quality, then merged.
- Replication reports: published as submitted (we don't gate-keep your data) as long as the template is followed and limitations are disclosed.
- New articles: longer review; expect a back-and-forth on structure and sourcing.
